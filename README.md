# タンク管理シミュレーションシステム (Tank Management System)

## 1. 概要
本プロジェクトは、工場やプラントにおける貯蔵タンクの監視・制御を想定した**「業務ロジック検証用シミュレーター」**です。
単なる数値の変化だけでなく、**異常検知、相互監視、リソース競合（修理枠の制限）**など、実務で発生しうる複雑なビジネスロジックをC#で実装しています。



## 2. 実装した主要ロジック（ビジネスロジック）
Unityの機能を「表示」に限定し、裏側の計算は純粋なC#ロジックとして堅牢に設計しました。

### ■ フェイルセーフ設計 (Fail-safe)
監視用タンク（Tank C）の故障検知時、システム全体の安全を確保するために稼働中のタンク（A/B）を強制停止させるインターロック機能を実装しています。

### ■ 相互補完稼働 (Redundancy)
タンクAとBが相互に状態を監視。一方が停止・故障した際、他方が自動で稼働を開始する冗長化ロジックを組み込み、システムの稼働率を維持します。

### ■ 排他制御（修理リソースの制限）
複数の故障が発生しても、修理担当者（リソース）は1人のみを想定。コルーチンを用いた非同期処理により、「一度に1つのタンクしか修理できない」排他制御を実装しました。

### ■ 確率的エラー生成
`Update` 毎に `Random.value` を評価し、0.1%の低確率で発生する突発的故障を再現。予期せぬエラーに対するシステムの挙動を検証可能です。



## 3. 技術的特徴
* **状態管理**: `enum`（列挙型）による状態定義を行い、不正な状態遷移を物理的に防止。
* **データ構造の最適化**: `[System.Serializable]` を用いた内部クラス設計により、各タンクのパラメータをカプセル化し、拡張性を確保。
* **非同期処理**: 修理工程に `IEnumerator`（コルーチン）を採用し、メインロジックを阻害しないノンブロッキングな処理を実現。

## 4. 開発環境
* **言語**: C#
* **プラットフォーム**: Unity 6.3 / Visual Studio 2026
* **主要コンポーネント**: TextMeshPro, UI Slider

---

## 5. 設計思想：なぜこの構成か
「Unityを単なるゲームエンジンとしてではなく、**業務ロジックの可視化ツール**として活用する」という方針で開発しています。
内部ロジックはフレームワークに依存しすぎない疎結合な設計を心がけており、将来的な **ASP.NET Core** などのバックエンド環境へのロジック移植を容易にしています。
